<h1>Visitor // Посетитель</h1>
<hr>
<h3>
Описывает операцию, выполняемую с каждым объектом из некоторой структуры. Паттерн Посетитель
позволяет определить новую операцию, не изменяя классы этих объектов.<br><p></p>
Класс c точки зрения ООП, это и есть описание данных и методов работы с ними. Класс
предоставляет методы доступа, скрывая сами данные от внешнего мира. В тоже время встречаются
ситуации, когда структуры данных достаточно сложны, а операции производимые над ними или
настолько разнообразны, что приводит к неразумности объединения данных и операций (т.к. это
размывает ответственность), или еще неизвестны на момент проектирования структур данных.
Встречаются и ситуации наоборот, когда для стабильной иерархии объектов требуется выполнять
разноплановые операции без возможности модифицировать код, описывающий иерархию объектов,
представляющую некоторую предметную область.
</h3>
